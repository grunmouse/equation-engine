# Инструкция по использованию движка для парсинга и вычисления выражений

Движок `equation-engine` позволяет парсить математические выражения (формулы) с операторами, функциями и переменными, преобразовывать их в постфиксную нотацию (ПОЛИЗ) и вычислять. Он основан на алгоритме сортировочной станции (Shunting-yard) и стековой машине.

Код состоит из модулей:
* `lexer.js` — токенизация;
* `shunting_yard.js` — преобразование в ПОЛИЗ;
* `handle.js` — промежуточная обработка конфликтов;
* `mashine.js` — вычисление;
* `index.js` — сборка.

Движок гибкий: вы можете определять свои операторы, функции, переменные и типы значений. Ниже — инструкция по использованию с примерами.

---

## 1. Основные понятия

* **Парсер** (`createParser`) — преобразует строку‑формулу в массив команд в ПОЛИЗ (постфиксной нотации).
* **Машина** (`createMashine`) — выполняет ПОЛИЗ с помощью стека. Возвращает массив, содержащий один или несколько результатов.
* **Токены** — литералы (числа, строки), переменные, операторы, функции.
* **ПОЛИЗ** — форма записи, в которой операторы идут после операндов. Например, `A B +` вместо `A + B`.

---

## 2. Импорт и настройка

Импортируйте движок:

```javascript
const eng = require('@grunmouse/equation-engine');
```

---

## 3. Создание парсера

Парсер токенизирует формулу и преобразует её в ПОЛИЗ. Создайте его с помощью функции `createParser(operators, functions, values, names)`.

### Параметры

* **`operators`** — объект с операторами.  
  Ключи — символы операторов (например, `+`, `*`).  
  Значения — объекты со следующими свойствами:
    * `order` — приоритет (число; чем меньше значение, тем выше приоритет).
    * `arity` — арность (число аргументов; по умолчанию — 2).
    * `fix` — тип оператора (`'prefix'`, `'infix'`, `'postfix'`); по умолчанию для бинарных операторов — `'infix'`.
    * `right` — `true` для правоассоциативных операторов (например, `**` для возведения в степень).
    * `sign` — символ операции, если он отличается от ключа (используется для создания перегруженных операторов, например, унарный минус).
    * `com` — команда для машины, если она отличается от ключа (используется для псевдонимов команд, например, `**` и `^` для возведения в степень).

* **`functions`** — объект с функциями.  
  Ключи — имена функций.  
  Значения — объекты с полем `arity` (арность). Допускается значение `-1`, которое означает, что арность будет определена во время выполнения.

* **`values`** — массив парсеров для литералов.  
  Каждый элемент — массив `[regex, type, parser]`, где:
    * `regex` — регулярное выражение для распознавания литерала.
    * `type` — тип значения (строка, например, `'number'`).
    * `parser` — функция для преобразования строки в значение (например, `parseFloat`). Необязательный параметр: если его нет, преобразование значения остаётся на стороне машины.

* **`names`** — регулярное выражение, определяющее допустимые имена переменных (например, `/[A-Za-z_]+/`).

### Пример создания парсера

```javascript
const starssetParser = eng.createParser(
    {
        '&': { order: 1 },  // Приоритет 1, бинарный оператор
        '^': { order: 2 }   // Приоритет 2, бинарный оператор
    },
    {},                   // Функции не определены
    [],                   // Специальные литералы не определены
    /[A-Za-z_А-Яа-яЁё]+/  // Имена переменных (включая русские буквы)
);
```

## 4. Создание машины

Машина выполняет ПОЛИЗ. Создайте её с помощью функции `createMashine(commands, getVariable, valueParsers = {})`.

### Параметры

* **`commands`** — объект с командами.  
  Ключи — строковые имена команд.  
  Значения — функции, которые выполняют вычисления в соответствии с арностью команды.

  **Сигнатура:**
  ```javascript
  (...operands, env) => value | Error
  ```
  количество аргументов на один больше арности команды, в последнем агрументе предана ссылка на текущее окружение

  **Пример:**
  ```javascript
  {
      '+': (a, b) => a + b,
      'sin': (a) => Math.sin(a),
      'VAR': (name) => getVariable(name) // Использует внешнюю функцию
  }
  ```

* **`getVariable`** — функция для доступа к переменным.  

  **Сигнатура:**
  ```javascript
  (name, env, command) => value | Error
  ```

  **Параметры:**
  * `name` (строка) — имя переменной из ПОЛИЗ‑кода (например, `'x'` или `'user_data'`).
  * `env` (объект) — текущее окружение (содержит переменные и их значения).
  * `command` (объект) — внутреннее представление команды (например, `{ token: "variable", name: "x", ... }`). Используется для расширенной логики при необходимости низкоуровневого доступа.

  **Пример реализации:**
  ```javascript
  (name) => {
      if (env.variables[name] !== undefined) {
          return env.variables[name];
      }
      throw new Error(`Variable "${name}" not found`);
  }
  ```

* **`valueParsers`** (опционально) — объект парсеров для обработки литеральных значений.  
  * **Ключи** — строковые названия типов (например, `'number'`, `'string'`).
  * **Значения** — функции для преобразования сырого значения в нужный тип.

  **Сигнатура парсера:**
  ```javascript
  (value, env, command) => processedValue
  ```

  **Параметры:**
  * `value` (любой тип) — значение, вычисленное на этапе парсинга, либо строка исходного кода литерала (если на этапе парсинга преобразование не выполнялось).
  * `env` (объект) — текущее окружение.
  * `command` (объект) — внутреннее представление команды (например, `{ token: "literal", type: "number", str: "5", value: 5, ... }`). Используется для расширенной логики.

  **Пример:**
  ```javascript
  {
      number: (val) => parseFloat(val),           // Преобразует строку в число
      string: (val) => val.replace(/'/g, ''),      // Убирает кавычки
      json: (val) => JSON.parse(val)              // Разбирает JSON‑строку
  }
  ```

### Пример полной инициализации

```javascript
const vm = createMashine(
    // Определяем операторы и функции
    {
        '+': (a, b) => a + b,
        'if': (condition, trueBranch, falseBranch) => condition ? trueBranch : falseBranch
    },

    // Логика доступа к переменным
    (name, env) => {
        return env.memory[name] || env.globals[name];
    },

    // Парсеры для литералов
    {
        number: (val) => parseInt(val, 10),
        bool: (val) => val.toLowerCase() === 'true'
    }
);
```

## 5. Использование

1. **Парсинг.** Выполните `let poliz = parser(строка_формулы);`. В результате вы получите массив объектов — команды в ПОЛИЗ.
2. **Вычисление.** Выполните `let result = mashine(poliz, env);`. Функция вернёт массив, который представляет собой состояние стека на момент остановки машины. Если вычисление завершилось одним значением, оно будет единственным элементом в этом массиве.

Параметр `env` — окружение. Оно передаётся:
* в парсеры литералов;
* в функцию получения переменных;
* в реализации команд.

### Полный пример функции вычисления

```javascript
function evalStarsset(str, context) {
    let p = starssetParser(str);
    let res = starssetMashine(p, context);  // Передача контекста
    return res;
}

let context = { currentUser: 'admin', debug: true };
let result = evalStarsset('Ursa_Major & Orion', context);  // Пересечение множеств звёзд
```

### Второй пример: сферические вычисления

Парсер и машина для векторов, матриц и геометрии. Поддерживаемые операторы: `+`, `-`, `*`, `/`, `**` (степень), `.` (скалярное произведение), унарный `-` (отрицание). Поддерживаемые функции: `cross`, `abs`, `ort`, `angle`, `circle3P` и другие.

```javascript
function evalSpheric(str) {
    let p = sphericParser(str);
    let res = sphericMashine(p);
    return res;
}

// Пример использования:
let vec = evalSpheric('A + B * 2');  // Векторы A и B из getVariable
```
